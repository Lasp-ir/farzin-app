# ♟️ Farzin Project - Development Roadmap

## 🗄️ فاز ۱: راه‌اندازی و تزریق دیتابیس (سیستم خانه)
- [ ] دریافت آخرین کدهای سرور: اجرای `git pull` در مسیر پروژه برای دریافت ساختار جدید از Web Editor.
- [ ] نصب وابستگی‌ها: اجرای `npm install` در مسیر پوشه `backend`.
- [ ] ساخت دیتابیس لوکال: اجرای `npx prisma db push` برای متولد شدن `dev.db`.
- [ ] آماده‌سازی دیتا: اکسترکت فایل Lichess Puzzles و قرار دادن فایل `lichess_puzzles.csv` در پوشه `backend`.
- [ ] تزریق ۳.۵ میلیون پازل: اجرای `node importPuzzles.js` و صبر برای اتمام عملیات.
- [ ] تست سلامت API: اجرای `npm run dev` و بررسی خروجی روت `/api/puzzles/debug`.

## 🧩 فاز ۲: توسعه فرانت‌اند پازل‌ها (UI/UX)
- [ ] اتصال API: ساخت سرویس‌های Axios/Fetch در فرانت‌اند برای فراخوانی پازل‌های روزانه، ریتینگ‌دار و موضوعی از بک‌اند.
- [ ] پیاده‌سازی صفحه حل پازل: کانفیگ `react-chessboard` برای رندر پویای FEN دریافتی.
- [ ] منطق اعتبارسنجی: هندل کردن حرکات کاربر و مقایسه آن با فیلد Moves در دیتابیس.
- [ ] فیدبک بصری پازل: اضافه کردن انیمیشن، صدا و رنگ‌بندی مهره‌ها برای حرکات درست و غلط.
- [ ] سیستم امتیازدهی: ساخت روت‌های آپدیت برای تغییر `puzzleRating` و `puzzleStreak` کاربر بعد از حل هر پازل.

## 📊 فاز ۳: پنل کاربری و مدیریت (Dashboard)
- [ ] داشبورد آمار کاربر: طراحی UI برای نمایش نمودار پیشرفت، بالاترین استریک (Max Streak) و وضعیت سهمیه‌ها.
- [ ] مدیریت سهمیه (Quotas): اتصال UI فرانت‌اند به محدودیت‌های روزانه بک‌اند (Mistakes, Themes, Rush) و سیستم ریست زمان‌دار.
- [ ] مدیریت اکانت‌ها: پیاده‌سازی بخش تنظیمات برای همگام‌سازی اکانت‌های Lichess و Chess.com.

## 🤖 فاز ۴: ارتقاء و ادغام موتور تحلیل (Analysis Engine)
- [ ] اتصال انجین: برقراری ارتباط بین فرانت‌اند و منطق تحلیل بک‌اند.
- [ ] پیوسته بودن تحلیل: فعال نگه داشتن انجین و نوار ارزیابی (Eval Bar) بلافاصله پس از اتمام حرکات تئوری (Book Moves).
- [ ] منطق "Miss": پیاده‌سازی طبقه‌بندی "Miss" با رنگ قرمز روشن برای حرکاتی که فرصتِ استفاده از اشتباه حریف را از دست می‌دهند.
- [ ] فیلتر حرکات عالی: اصلاح منطق "Great Move" جهت حذف حرکات اجباری (Forced moves) و برجسته کردن تک‌حرکت‌های طلایی.
- [ ] توسعه "See Why?": ادامه دادن واریانت‌های پیشنهادی تا زمان مشخص شدن دقیق برتری یا ضعف در پوزیشن (جلوگیری از پیشنهاد تک‌حرکتی).
