# نقشه راه توسعه لابراتوار آنالیز فرزین

## فاز ۱: ارتقای هسته‌ی مرکزی (AnalysisBoard) برای هضم PGN
قبل از دست زدن به صفحه Setup، باید `AnalysisBoard` رو هوشمند کنیم. باید تو منطق لود اولیه‌اش (`useEffect`) یه پارسر (Parser) بنویسیم که:
- [ ] اگر ورودی PGN بود، تمام حرکات رو یکی‌یکی روی تخته‌ی مجازی (`chess.js`) اجرا کنه.
- [ ] گرافِ درختی (Tree) حرکات رو به صورت خودکار بسازه.
- [ ] نام بازیکن‌ها، ریتینگ‌ها و نتیجه بازی رو از متادیتا (Metadata) استخراج کنه و تو کادر `EditablePlayer` نمایش بده.

## فاز ۲: استانداردسازیِ پُل ارتباطی (Payload Protocol)
باید یک قرارداد بین `AnalysisSetup` و `AnalysisBoard` تنظیم کنیم. هر روشی که در Setup انتخاب بشه، در نهایت باید یک پکیج استاندارد (Object) شبیه به این تولید کنه و بفرسته:
- [ ] پیاده‌سازی ساختار استاندارد انتقال دیتا:
  ```json
  {
    "type": "FEN", // یا "PGN"
    "data": "...", // رشته FEN یا PGN کامل
    "meta": {
      "whiteName": "...",
      "whiteElo": "...",
      "blackName": "...",
      "blackElo": "..."
    }
  }
